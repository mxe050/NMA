  <span class="toggle">▼</span>

<div class="section-body">

<div class="subsection">
<h3>1.1 ネットワークメタアナリシス（NMA）とは</h3>
<p>
<span class="term" tabindex="0">ネットワークメタアナリシス（NMA）<span class="term-popup">複数の治療法を同時に比較するため、直接比較と間接比較のエビデンスを1つの統計モデルで統合する手法。直接比較試験が存在しない治療ペア間でも効果を推定できる。</span></span>は、複数の治療法を同時に比較し、<strong>直接比較のエビデンスと間接比較のエビデンスを1つの統計モデルで統合</strong>する手法です。これにより、直接比較試験が存在しない治療ペア間でも効果を推定できます。
</p>

<div class="cite-box">
<strong>引用元:</strong> Izcovich et al. BMJ 2023;381:e074495, p.1: "Network meta-analysis (NMA) allows assessing the comparative effectiveness of multiple interventions by combining direct and indirect evidence in one statistical model, resulting in estimates of effect comparing every pair of interventions included in the network—even if they have not been compared directly in trials."
</div>

<p>
しかし、NMAの包括性は同時に複雑性の増大を意味します。より多くのデータソース、間接的な関連性、そして<span class="term" tabindex="0">推移性<span class="term-popup">間接比較を構成する直接比較間で、効果修飾因子の分布に系統的な差がないという仮定。A対CのCとB対CのCが同等であること。</span></span>や<span class="term" tabindex="0">整合性<span class="term-popup">同じ比較について、直接推定値と間接推定値が一致するという仮定。</span></span>といったNMA特有の仮定の検証が必要です。
</p>

<div class="cite-box">
<strong>引用元:</strong> ネットワークメタ分析GRADE評価詳細レポート, p.2: "NMAの強みである包括性そのものが、単純なメタアナリシスよりも厳密でニュアンスに富んだ確実性評価アプローチを必要とするのです。"
</div>
</div>

<div class="subsection">
<h3>1.2 なぜGRADEアプローチが必要か</h3>
<p>
NMAの結果の信頼性、すなわち<span class="term" tabindex="0">エビデンスの確実性<span class="term-popup">効果の推定値が真の効果に近いとどの程度確信できるかの度合い。高・中・低・非常に低の4段階。</span></span>を評価することが極めて重要です。GRADEアプローチは、<span class="cert c-high">高</span>・<span class="cert c-mod">中</span>・<span class="cert c-low">低</span>・<span class="cert c-vlow">非常に低</span>の4段階で評価します。
</p>

<div class="cite-box">
<strong>引用元:</strong> Izcovich et al. BMJ 2023, p.1: "Assessing the certainty of the evidence (also known as quality of the evidence, and confidence in the estimates of effects) from NMAs is crucial for interpreting those estimates and moving from evidence to decision."
</div>

<div class="key-point">
<strong>GRADEの核心:</strong> 単に確実性のラベルを付与するだけでなく、評価を特定のドメインとステップに分解し、評価者は自らの判断とその根拠を明確に記述することが求められます。この透明性が、NMA結果の信頼性と実用性にとって極めて重要です。
<br><span style="font-size:.85em;color:#555;">（ネットワークメタ分析GRADE評価詳細レポート, p.3）</span>
</div>
</div>

<div class="subsection">
<h3>1.3 NMA-GRADEの階層的アプローチ：4段階の評価</h3>
<p>NMAにおけるGRADE評価は、以下の4段階を順番に進めます：</p>

<div class="flow-h">
  <div class="flow-box" style="background:var(--step-a)">A: 直接推定値<br>の確実性評価</div>
  <div class="flow-arrow">→</div>
  <div class="flow-box" style="background:var(--step-b)">B: 間接推定値<br>の確実性評価</div>
  <div class="flow-arrow">→</div>
  <div class="flow-box" style="background:var(--step-c)">C: NMA推定値<br>の確実性評価</div>
  <div class="flow-arrow">→</div>
  <div class="flow-box" style="background:var(--step-d)">D: 最良推定値<br>の選択</div>
</div>

<div class="cite-box">
<strong>引用元:</strong> Izcovich et al. BMJ 2023, Fig 1 および Supplementary table 1: 評価プロセスはA（直接）→B（間接）→C（NMA）→D（最良推定値選択）の順で進行。各比較に対して12-13の判断が必要。
</div>

<p>
この順序の理由：NMA推定値は直接推定値と間接推定値を統合したものであり、その質は構成要素の質に依存します。直接・間接エビデンスの評価を先に行うことで、基礎的な構成要素が精査されることが保証されます。
</p>

<div class="warning-point">
<strong>重要原則:</strong> NMA推定値が常に最良の推定値とは限りません。非整合性が検出された場合、直接推定値や間接推定値の方がより高い確実性を持つことがあります。
<br><span style="font-size:.85em;">（Izcovich et al. BMJ 2023, p.6-7, Step D; Brignardello-Petersen et al. JCE 2019;108:77-85, Section 8）</span>
</div>
</div>

<div class="subsection">
<h3>1.4 作業量の課題と効率化の必要性</h3>
<p>ネットワークに含まれる介入（ノード）の数をnとすると、比較数は n(n-1)/2 です。</p>
<table>
<tr><th>介入数</th><th>比較数</th><th>判断数（1比較12-13）</th><th>出典の例</th></tr>
<tr><td>4</td><td>6</td><td>~78</td><td>―</td></tr>
<tr><td>10</td><td>45</td><td>~585</td><td>Brignardello-Petersen 2018, p.37</td></tr>
<tr><td>57</td><td>1,596</td><td>~20,748</td><td>Izcovich 2023, p.1 (COVID-19 NMA)</td></tr>
</table>

<div class="cite-box">
<strong>引用元:</strong> Izcovich et al. BMJ 2023, p.1: "a network with four interventions, there are six comparisons for which to assess the certainty of evidence, whereas a network with 10 interventions has 45 comparisons. To complete the GRADE certainty of the evidence assessment, raters need to make between 12 and 13 judgments for every comparison."
</div>

<p>この課題に対処するため、本ガイドで紹介するルール設定・自動化・効率的な手順設計が不可欠です。</p>
</div>

// ==================== セクション開閉 ====================
function toggleSec(header){
  const body=header.nextElementSibling;
  const t=header.querySelector('.toggle');
  if(body.classList.contains('hidden')){
    body.classList.remove('hidden');t.textContent='▼';header.classList.remove('collapsed');
  }else{
    body.classList.add('hidden');t.textContent='▶';header.classList.add('collapsed');
  }
}

// ==================== ナビゲーション ====================
function goSec(id){
  const el=document.getElementById(id);
  if(!el)return;
  const h=el.querySelector('.section-header');
  const b=el.querySelector('.section-body');
  if(b&&b.classList.contains('hidden')){b.classList.remove('hidden');h.querySelector('.toggle').textContent='▼';h.classList.remove('collapsed');}
  el.scrollIntoView({behavior:'smooth',block:'start'});
}

// ==================== タブ ====================
function showTab(ev,tabId){
  const c=ev.target.closest('.tab-container');
  c.querySelectorAll('.tab-btn').forEach(b=>b.classList.remove('active'));
  c.querySelectorAll('.tab-content').forEach(t=>t.classList.remove('active'));
  ev.target.classList.add('active');
  document.getElementById(tabId).classList.add('active');
}

// ==================== 折りたたみ詳細 ====================
function togDetail(btn){
  const c=btn.nextElementSibling;
  c.classList.toggle('show');
  btn.textContent=c.classList.contains('show')?'▲ 詳細を閉じる':'▼ 詳細を表示';
}

function renderGlossary(){
  const c=document.getElementById('glossary-list');
  glossary.forEach((g,i)=>{
    const d=document.createElement('div');
    d.className='gl-item';
    d.innerHTML=`
      <div class="gl-header" onclick="toggleGl(${i})">
        <span><span class="gl-term">${g.term}</span><span class="gl-en">(${g.en})</span></span>
        <span class="gl-arrow" id="gla-${i}">▶</span>
      </div>
      <div class="gl-body" id="glb-${i}">
        <div class="gl-def"><strong>定義:</strong> ${g.def}</div>
        <div class="gl-detail"><strong>詳細:</strong> ${g.detail}</div>
        <div class="gl-cite"><strong>出典:</strong> ${g.cite}</div>
      </div>`;
    c.appendChild(d);
  });
}
function toggleGl(i){
  const b=document.getElementById('glb-'+i);
  const a=document.getElementById('gla-'+i);
  if(b.classList.contains('open')){b.classList.remove('open');a.style.transform='rotate(0deg)';}
  else{b.classList.add('open');a.style.transform='rotate(90deg)';}
}
